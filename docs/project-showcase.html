Seguridad — OWASP LLM Top 10

Mitigaciones implementadas y testeadas (39 tests de seguridad corren en CI). pip-audit como gate bloqueante en cada push.

LLM01 — Prompt Injection

Delimitadores de contexto

Template con etiquetas <context>…</context> que señalan al modelo que ese contenido es dato, no instrucción.

LLM04 — Model DoS

Límites explícitos de ingesta

Validación antes de indexar: máx. 2 000 chars por query, 25 MB / 300 páginas por PDF. Magic bytes verificados.

LLM06 — Data Disclosure

PII redactado en logs

redact_pii=true en la capa de observabilidad. Hashing de queries; ningún dato personal se almacena en claro.

Corpus Poisoning

Sanitización antes de indexar

Caracteres de control eliminados antes del chunking para mitigar secuencias maliciosas embebidas en PDFs.

Quickstart

Prerequisitos: Python 3.11 y Ollama instalado.

# 1. Clonar e instalar dependencias
git clone https://github.com/victorlr94/mexico-immigration-rag.git
cd mexico-immigration-rag
python -m venv .venv && .venv\Scripts\Activate.ps1
pip install -e ".[dev]"

# 2. Descargar el modelo LLM (~4.9 GB, solo la primera vez)
ollama pull llama3.1:8b

# 3. Indexar el corpus de muestra (4 PDFs oficiales)
python scripts/ingest.py data/samples/*.pdf

# 4. Lanzar el asistente
streamlit run app/streamlit_app.py

# O en un solo comando con el Makefile:
make demo # instala + descarga modelo + indexa
make run # lanza Streamlit
